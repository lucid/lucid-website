About

Lucid's package repository contains additional packages that can be installed to provide a component of the desktop. It contains packages provided by the Lucid Desktop Project, as well as third-party packages. They provide additional themes, apps, translations, and other enhancements.

Although this directory is provided by the Lucid Desktop Project, be cautious when installing packages. Packages that have been marked as 'verified' have been checked, and are known to be free of malicious code. Unverified third-party packages can contain malicious code, and should be approached with caution. Also, make sure you verify the checksum. Third-party packages are hosted on remote servers, allowing the package creator to add malicious code to the package after it has been verified. The checksum is generated when the package is first submitted, and does not change.

Getting your package listed

The first thing you should do is upload your package to a server. If you do not want to host the package yourself, it is recommended that you use Omploader to host the package. Websites like Rapidshare and Mediafire will not work, because they do not allow direct access to the file.

Once your package has been hosted, you can fill out this form. After submission, it will be posted on the site.

There are some simple rules for packages listed here. If a package does not follow these rules, it will be removed.

  1. The package cannot contain malicious code.
  2. The package should not conflict with other packages in the directory.
  3. If the package contains an app, the app must not declare global variables.
There are also some general guidelines for packages. These are not enforced as heavily as rules, but your package could be removed for one of these reasons.
  1. If the package contains an app:
    • The app should not use IDs in DOM Nodes
    • The app should not insert DOM Nodes directly into the document
  2. The package should remove itself completely when uninstalled, and should not leave traces of itself afterword. The exception to this would be if the package left behind configuration or other information.
  3. The package should include all the necessary files for it to be used (including images, swf files, javascript files, etc.). Loading files from an external server should only be done when necessary.
